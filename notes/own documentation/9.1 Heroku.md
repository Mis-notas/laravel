## 1. Tener una cuenta en Heroku
## 2.  Instalar Heroku
### Heroku CLI
Para configurar la CLI en /usr/local/lib/heroku y /usr/local/bin/heroku, ejecute el siguiente script.
~~~TERM
sudo curl https://cli-assets.heroku.com/install.sh | sh
~~~

Verificar instalación:

~~~TERM
heroku --version
~~~

[Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli)
[Getting Geroku](https://devcenter.heroku.com/articles/getting-started-with-laravel)

~~~PHP
sudo snap install heroku --classic
~~~

## 3. Crear el archivo Profile 
- P en mayúscula y sin extensión. 

~~~TERM
touch Procfile
~~~

Editar el archivo

~~~TERM
nano Procfile
~~~

Se copia y pega el siguiente contenido: 

~~~TERM
web: vendor/bin/heroku-php-apache2 public/
~~~

Ahora se agrega en git

~~~TERM
git add --all
~~~
~~~TERM
git commit -m "Procfile agreed"
~~~

## 4. Compilar assets -> Para producción

~~~TERM
npm run production
~~~

``` ad-info
title: Git
Se agrega nuevamente a git
```

## 5. Creamos la aplicación desde el terminal 

```term
heroku create name
```
~~~ ad-note
title: configuración
Esto me configura automaticamente el servidor web y un github del proyecto.
~~~
## 6. Desplegamos la aplicación 
~~~TERM
heroku config:set APP_KEY=... 
~~~

~~~PHP
php artisan key:generate --show
~~~
Esto nos dejaría visualizar la llave que de aplicación que ya usamos.

### Push Heroku

~~~TERM
git push heroku develop:master
~~~

~~~ ad-note
title: info
Develop es la rama en donde estoy parado y la rama master es la del servidor de heroku
~~~

### Abrir el proyecto 

~~~TERM
heroku open
~~~


## 7. Base de datos
### Add-ons
Heroku Postgres
Una alternativa gratuita para usar la base de datos de postgres en el proyecto.
## 7.1 Credenciales de la base de datos
Entrando en settings del  recurso de base de datos creado, en la opción de view credentials las podemos encontrar y pasar  a las variables de configuración. 
### 7.2 Config Vars
Son  las variables de entorno que se necesita para que el aplicativo funcione, la rauta es: 
Home->Settings->Reveal Config Vars
En este espacio se debe agregar todas las variables que requiera el proyecto (base de datos, correos, etc)

## 7.3 Consola de Heroku 
~~~TERM
heroku run bash
~~~
~~~PHP
php artisan migrate --seed
~~~

~~~ ad-note
title: Seed
Los seeders no se ejecutan en un entorno de producción ya que esto es información falsa.
~~~
En caso de que el aplicativo de error por estos seeders de tipo que no existe la libretia Faker, se debe ir al proyecto en su sección requiere-dev, y se pasa a la sección require.
Posteriormente se ejecuta el comando
~~~TERM
composer update
~~~
Se hace un git push y se ejecuta de nuevo la migracion con su seeders.

Para los errores que hay de https en el aplicativo se debe ingresar en la siguiente ruta:
app->Providers->AppService->Provider.php y colocar en su función boot.
~~~PHP
public function boot()  
{  
    if(config('app.env') === 'production'){  
        URL::forceScheme('https');  
    }  
}
~~~
